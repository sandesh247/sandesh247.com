\resumetitle{@ Finacus \hfill \small{\url{http://www.finacus.co.in/}}}
	\begin{resumedescription}
		\item[Evolution]
			Evolution was the framework on which the all the products of the organization were built. This framework consisted of a controls library, an OR mapping framework, code generation templates, file conversion utilities etc.

			\begin{resumesubdescription}
				\item[OR Framework]
				\begin{itemize*}
					\item single entry point to all database operations, based on the front controller approach
					\item supported auditing facilities, all changes to the database were logged
					\item used optimistic concurrency control, users would be alerted if they were working on a stale version
					\item allowed developers to manipulate a database row with minimal knowledge (deserves explanation)
				\end{itemize*}
			\end{resumesubdescription}

			\begin{resumesubdescription}
				\item[Controls]
				\begin{itemize*}
					\item controls to perform basic input, lookup--based selection and picklists
					\item optional rendering of controls based on user authorization
					\item runtime control over number/types of controls being rendered based on configuration
					\item controls for multiline (excel--like) editing of values
				\end{itemize*}
			\end{resumesubdescription}
			
			\begin{resumesubdescription}
				\item[File conversion API]
				\begin{itemize*}
					\item supported Excel, delimited text (csv), and databases
					\item XML based configuration, source and destination columns were configurable at runtime
					\item supported custom transformations mid--flow
				\end{itemize*}
			\end{resumesubdescription}
			
		\item[AML]
			AML (Anti money laundering) was the second product which was being made on the Evolution framework. I led the product development until I left the organization. During this period, some of my responsibilties were --
			\begin{itemize*}
				\item requirements gathering from the customers
				\item designing the data model
				\item design a framework for workflows to support escalation of tasks within an organization
			\end{itemize*}

		\item[Administrative Responsibilities]
			\begin{itemize*}
				\item set up source control policies, with some customization of SVN
				\item took interviews for new candidates in the organization
				\item evaluated 3rd party frameworks for use in the organization
			\end{itemize*}
	\end{resumedescription}

\resumetitle{@ Engineering}
\begin{resumedescription}
		\item[TDDB]
			TDDB was a final year project undertaken by three groups belonging to three different colleges. TDDB is an acronym for \textbf{T}he \textbf{D}istributed \textbf{D}ata\textbf{B}ase. It was a functional RDBMS, with support for many SQL statements, like SELECT, INSERT, CREATE TABLE etc. The responsibilities of the three colleges were to develop the storage engine, the parser, and the distribution mechanism. Our group developed the parser and the gui interface for the engine. More information can be had at \url{http://tddb.sourceforge.net/}.

My responsibilities in this project were :
			\begin{itemize*}
				\item design the parser in flex/bison 
				\item design C++ data structures to interact with the storage engine
				\item optimization of queries
				\item development of a front--end in QT
			\end{itemize*}

\item[OXEON]
			OXEON was intended to be a corporate mailing solution. It consisted of a mail server and GUI client. It used cryptographic libraries from the `Bouncy Castle' project to provide security.

\item[Book Xchange]
			This was a website developed as a part of the curriuculum, which intended to facilitate exchange of books among students. Students could put up books for exchange/sell with an asking price, and could also request books. However, no mechanism of payment was provided.

\end{resumedescription}
